# Studio Brain Guide

## What the Studio Brain Does
The Studio Brain runs the `/start` command. It transforms a raw idea into a fully configured project through a structured 7-phase sequence.

## The 7 Phases
1. **Identity** — Who are you? Developer or idea-first founder?
2. **Concept Interview** — 5-8 questions about the idea, user, and constraints
3. **Concept Challenge** — Honest pressure-test before any documents are written
4. **Track Detection** — Mapping the idea to the right studio configuration
5. **Document Generation** — All project documents, with approval at each step
6. **Sprint Planning** — Full sprint roadmap from foundation to launch
7. **Handoff** — Configured project, first tasks, ready to build

## For Non-Developers
The Studio Brain uses plain English throughout. Technical concepts are explained when they arise. The Architecture Director recommends a tech stack based on the project type — you don't need to know what to choose.

## For Developers
The sequence is the same but questions are direct and concise. You can override any technology recommendation. Stack decisions are yours — the Architecture Director provides reasoning and trade-offs.

## The Concept Challenge
This is the most valuable phase. Before writing any document, the Studio Brain will challenge your concept across:
- Market reality (is this crowded?)
- User specificity (who exactly?)
- Retention mechanic (what brings them back?)
- Scope realism (buildable in reasonable time?)
- Monetization viability (will they pay?)

The output is a Go / Refine / Pivot recommendation. The studio will not proceed to documents on a Pivot concept without explicit agreement on a direction.

## Dynamic Track Generation
If your idea doesn't fit any existing track, the Track Architect generates a custom one during the `/start` sequence. This is fully automated — you'll be asked to approve the generated agents, skills, and template before they're written.

## Running /start Again
Running `/start` on an existing project will detect the existing CLAUDE.md and offer to update the configuration rather than starting from scratch.
